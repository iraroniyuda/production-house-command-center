# AI Working Agreement

Status: Approved  
Owner: Roni / Product & Engineering  
Last Reviewed: 2026-07-22  
Applies To: All AI Contributions  
Related Documents:

## Before Coding

AI MUST output:

1. Understanding
2. Referenced specifications
3. Assumptions
4. Affected modules
5. Business rules
6. Data impact
7. API impact
8. Security impact
9. Transaction boundary
10. Proposed files
11. Test plan
12. Documentation updates
13. Risks

## During Coding

- Keep scope narrow.
- Preserve naming from glossary.
- Reference rule IDs.
- Avoid unrelated refactor.
- Add failure-path tests.
- Keep code production-ready.
- Avoid placeholder security.

## After Coding

AI MUST summarize:

- files changed;
- behavior added;
- tests added;
- migration;
- API contract;
- remaining gaps;
- documents updated;
- commands to verify.
